
## Постановка задачи
Лабораторная работа 3 (Система логирования)

Логи - это записи о событиях в программе (ошибки, предупреждения, информационные сообщения).

Система логирования в этом контексте - это гибкий механизм, который:

Фильтрует сообщения (например, только ошибки)

Рассылает их в разные места (файл, консоль, сеть и т.д.)


## class ILogFilter 
```cpp
class ILogFilter {
public: 
    virtual bool match(const std::string& text) const = 0;
    virtual ~ILogFilter() = default;
};

```

Определяет какие методы должны быть у класс наследника, в данном случае метод match определяет подходит ли сообщение под условие фильтра. 
И виртуальный деструктор по умолчанию. Зачем нужен: Чтобы при удалении объекта через указатель на интерфейс правильно вызывались деструкторы всех наследников.

**Агрегация** (слабая связь, объекты существуют независимо):
- `Logger` хранит указатели на фильтры и обработчики, созданные вне класса
- Объекты `error_filter`, `console_handler` и др. создаются в `main()` и передаются в логгер

**Композиция** (сильная связь, объекты создаются внутри):
- `SocketHandler` создает объект `Socket` внутри себя
- `FileHandler` создает `std::ofstream` внутри метода `handle()`
- `ReLogFilter` создает `std::regex` внутри себя

**Коротко:** Логгер **агрегирует** фильтры и обработчики, а сами обработчики используют **композицию** для внутренних объектов (сокетов, файловых потоков).
